## Custom Slash Commands

*Reusable slash commands for common workflows. Place these in `.claude/commands/` directory.*

### Code Review
```markdown
<!-- .claude/commands/review.md -->
Review the code in $ARGUMENTS for:
- Security vulnerabilities
- Performance issues
- Code style and best practices
- Potential bugs

Provide actionable feedback with specific line references.
```

### Generate Tests
```markdown
<!-- .claude/commands/test.md -->
Generate comprehensive unit tests for $ARGUMENTS:
- Cover edge cases
- Include positive and negative test cases
- Follow existing test patterns in the codebase
- Use the project's testing framework
```

### Explain Code
```markdown
<!-- .claude/commands/explain.md -->
Explain the code in $ARGUMENTS:
- What does it do?
- How does it work?
- What are the key components?
- Any potential issues or improvements?
```

### Git Commit
```markdown
<!-- .claude/commands/commit.md -->
Review staged changes and create a commit with:
- Clear, conventional commit message
- Appropriate scope and type
- Brief description of changes
```

## Hooks & Automation

*Pre and post hooks to automate workflows with Claude Code.*

### Pre-commit Hook Example
```json
{
  "hooks": {
    "pre-commit": {
      "command": "npm run lint && npm run test",
      "description": "Run linting and tests before committing"
    }
  }
}
```

### Auto-format Hook
```json
{
  "hooks": {
    "post-file-edit": {
      "command": "prettier --write $FILE",
      "description": "Auto-format files after editing"
    }
  }
}
```

### Notification Hook
```json
{
  "hooks": {
    "post-task-complete": {
      "command": "notify-send 'Claude Code' 'Task completed!'",
      "description": "Desktop notification when task completes"
    }
  }
}
```

## Tips & Tricks

*Power user techniques for getting the most out of Claude Code.*

### CLAUDE.md Best Practices
- Keep project context in `CLAUDE.md` at repository root
- Include coding standards, architecture decisions, and common patterns
- Document project-specific commands and workflows
- Update regularly as the project evolves

### Effective Prompting
- Be specific about file paths and function names
- Provide context about the broader goal
- Use `/compact` to reduce context when hitting limits
- Reference specific line numbers when discussing code

### Performance Tips
- Use `--model` flag to switch between models for different tasks
- Enable `--dangerously-skip-permissions` for trusted automation (use carefully!)
- Use `/clear` to reset context when starting new tasks
- Leverage parallel tool calls by structuring requests appropriately

### Keyboard Shortcuts
- `Ctrl+C` - Cancel current operation
- `Ctrl+D` - Exit Claude Code
- `Tab` - Autocomplete file paths
- `Up/Down` - Navigate command history
