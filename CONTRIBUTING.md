# Contributing to Awesome Claude Code

Thank you for your interest in contributing to Awesome Claude Code! This document provides guidelines for contributions.

## How to Contribute

### Adding a New Resource

1. **Fork** this repository
2. **Add** your resource to the appropriate section in `README.md`
3. **Follow** the entry format (see below)
4. **Submit** a pull request

### Entry Format

Each entry should follow this format:

```markdown
- [Resource Name](https://url.com) - Brief description in one line
```

**Example:**
```markdown
- [mcp-server-redis](https://github.com/example/mcp-server-redis) - Redis database integration for Claude Code
```

### Section Guidelines

| Section | Content Type |
|---------|-------------|
| Official Resources | Anthropic official docs, repos, announcements |
| Tutorials & Guides | How-to articles, setup guides, best practices |
| MCP Servers | Model Context Protocol server implementations |
| Custom Slash Commands | Reusable command templates with code |
| Hooks & Automation | Hook examples and automation scripts |
| Tips & Tricks | Power user tips, keyboard shortcuts, workflows |
| Videos & Podcasts | Video tutorials, podcast episodes |
| Community Projects | Open source projects and tools |
| IDE Integrations | Editor plugins and extensions |

## Pull Request Process

1. **Ensure** your addition follows all guidelines
2. **Update** the Table of Contents if adding a new section
3. **Check** for broken links before submitting
4. **Write** a clear PR title and description
5. **Wait** for review from maintainers

### PR Title Format

```
Add [Resource Name] to [Section Name]
```

**Examples:**
- `Add mcp-server-mongodb to MCP Servers`
- `Add Claude Code tutorial video to Videos & Podcasts`
- `Fix broken link in Official Resources`

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Help maintain quality standards
- Report broken links or outdated resources

## Questions?

If you have questions about contributing, please:
1. Check existing issues for answers
2. Open a new issue with your question
3. Tag it with `question` label
